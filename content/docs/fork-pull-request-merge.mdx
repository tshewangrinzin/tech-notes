In this guide, you'll learn how to contribute to a project by forking it, making changes, and submitting a pull request to merge those changes back into the original repository.

<Callout title="Why Fork & PR?">
Forking allows you to freely experiment with changes without affecting the original project. Pull Requests (PRs) are the bridge that lets you propose those changes for review and eventual merging into the main project.
</Callout>

---

### Alternative: Making Changes Locally

If you prefer working on your own machine, you can clone your forked repository and make changes locally.

<Steps>
<Step title="Clone Your Fork">
Copy the SSH or HTTPS URL of your forked repository and run the following command in your terminal:

```bash
git clone https://github.com/your-username/repository-name.git
```
</Step>

<Step title="Navigate & Edit">
Go into the project folder and open the file you want to change (e.g., `README.md`) in your favorite editor.

```bash
cd repository-name
# Open README.md and make your changes
```
</Step>

<Step title="Stage & Commit">
Once you've made your changes, stage them and create a commit.

```bash
git add .
git commit -m "commit message"
```
</Step>

<Step title="Push to GitHub">
Push your local changes back to your forked repository on GitHub.

```bash
git push origin main
```
</Step>
</Steps>

<Callout title="Next Step">
After pushing your changes locally, you can proceed to **Phase 3** to create a Pull Request on GitHub.
</Callout>

</Step>

<Step title="Open Contribute Dropdown">

![Step 12 screenshot](https://images.tango.us/workflows/a75ac8c6-a6e5-49e7-bcb6-3491ce6701a2/steps/dde0ff1d-68e1-4482-8bae-f4da3f95435e/2e5a9484-59f7-4c08-964c-4caa89c887d5.png?mark-x=90&mark-y=584&m64=aHR0cHM6Ly9pbWFnZXMudGFuZ28udXMvc3RhdGljL2JsYW5rLnBuZz9tYXNrPWNvcm5lcnMmYm9yZGVyPTQlMkNGRjc0NDImdz0zMDQmaD04MCZmaXQ9Y3JvcCZjb3JuZXItcmFkaXVzPTEw)

## Phase 4: Merging the Pull Request

Once the pull request is approved, it can be merged into the target branch.

<Steps>
<Step title="Initiate Merge">
As a maintainer (or if you have permissions), click the **Merge pull request** button.

![Step 16 screenshot](https://images.tango.us/workflows/a75ac8c6-a6e5-49e7-bcb6-3491ce6701a2/steps/d261b65e-eefb-4e37-9aee-1e02f4cacd8f/e40625c9-3794-4719-b7a3-52feb4c28353.png?mark-x=234&mark-y=762&m64=aHR0cHM6Ly9pbWFnZXMudGFuZ28udXMvc3RhdGljL2JsYW5rLnBuZz9tYXNrPWNvcm5lcnMmYm9yZGVyPTQlMkNGRjc0NDImdz0zMTYmaD04MCZmaXQ9Y3JvcCZjb3JuZXItcmFkaXVzPTEw)
</Step>

<Step title="Add Final Details">
You can add an extended description or notes for the merge if necessary.

![Step 17 screenshot](https://images.tango.us/workflows/a75ac8c6-a6e5-49e7-bcb6-3491ce6701a2/steps/f91d40fa-b10b-4701-b8b6-5e1529e2015e/f62c6c3d-544e-4d08-a797-b316c92c1099.png?mark-x=236&mark-y=960&m64=aHR0cHM6Ly9pbWFnZXMudGFuZ28udXMvc3RhdGljL2JsYW5rLnBuZz9tYXNrPWNvcm5lcnMmYm9yZGVyPTQlMkNGRjc0NDImdz0xMTMyJmg9MzQ0JmZpdD1jcm9wJmNvcm5lci1yYWRpdXM9MTA%3D)

<Step title="Confirm & Finalize">
Click **Confirm merge**. GitHub will now integrate the changes from your pull request into the project.

![Step 18 screenshot](https://images.tango.us/workflows/a75ac8c6-a6e5-49e7-bcb6-3491ce6701a2/steps/e873d3a2-842e-4876-bd6e-46fb03095bed/0b81aaa2-1b00-441b-8c81-290d090ebcb0.png?mark-x=234&mark-y=1396&m64=aHR0cHM6Ly9pbWFnZXMudGFuZ28udXMvc3RhdGljL2JsYW5rLnBuZz9tYXNrPWNvcm5lcnMmYm9yZGVyPTQlMkNGRjc0NDImdz0yNjYmaD04MCZmaXQ9Y3JvcCZjb3JuZXItcmFkaXVzPTEw)

</Steps>

<Callout type="info" title="Success!">
Your changes are now part of the main project. Don't forget to sync your fork with the upstream repository periodically to stay up to date!
</Callout>
